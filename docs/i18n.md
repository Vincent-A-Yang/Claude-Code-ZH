# Claude Desktop 汉化流程

## 目标

在不改业务逻辑的前提下，将 Claude Desktop 中残留英文界面替换为中文，尤其是开发者设置、MCP、extensions、plugins、Claude Code、gateway 配置等页面。

## 原则

1. 只替换完整字符串字面量。
2. 不替换变量名、函数名、CSS class、React component 名称、事件名或协议字段。
3. 每轮只处理一组明确页面，补丁后立即做语法检查。
4. 每次修改前备份原 JS 文件。
5. 不把已经修改过的 Claude app 文件提交到本仓库。

## 推荐步骤

1. 找到 Claude Desktop app 根目录。
2. 复制目标 JS chunk 到备份目录。
3. 用 `translations/zh-CN.sample.json` 作为起点建立自己的翻译表。
4. 先 dry-run：

```powershell
.\scripts\Patch-ClaudeI18n.ps1 `
  -ClaudeAppRoot "C:\Path\To\ClaudeApp" `
  -TranslationFile .\translations\zh-CN.sample.json `
  -DryRun
```

5. 确认命中列表合理后应用：

```powershell
.\scripts\Patch-ClaudeI18n.ps1 `
  -ClaudeAppRoot "C:\Path\To\ClaudeApp" `
  -TranslationFile .\translations\zh-CN.sample.json
```

6. 对修改过的 JS 执行 `node --check`。
7. 重启 Claude，检查目标页面和日志。

## 常见问题

- 如果 UI 仍然有英文，通常是相关字符串在新加载 chunk 中，继续搜索完整英文文案。
- 如果 `node --check` 失败，立刻恢复备份，不要启动 Claude 继续试。
- 如果某些英文来自远端数据、模型名称、协议字段或插件包，不建议硬替换。
